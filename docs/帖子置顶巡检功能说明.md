# 帖子置顶巡检功能说明

## 功能概述

`thread_pin_watch` 是一个独立的论坛帖子巡检模块，用来保证多个论坛频道中的“长期置顶帖”不会因为人工操作或 Discord 状态变化而掉置顶。

核心能力：
- 维护一个“需要长期保持为置顶帖”的列表
- 每 6 小时自动巡检一次
- 发现帖子掉置顶后自动恢复
- 每次巡检后可向指定频道发送报告
- 支持批量添加、全服扫描当前置顶帖、一键导入扫描结果
- 支持分页查看当前巡检列表与最近状态

---

## 模块启用

请确认 `config.json` / `config.example.json` 的 `cogs` 中包含：

```json
"thread_pin_watch": {
  "enabled": true,
  "description": "帖子置顶巡检功能"
}
```

---

## 命令入口

本模块统一使用一条 Slash 命令：

```text
/置顶巡检
```

通过不同的 `操作` 选项完成不同功能。

---

## 支持的操作

### 1. 添加单个帖子
```text
/置顶巡检 操作:添加 帖子:<论坛帖子>
```
- 如果不填写 `帖子`，默认使用当前所在帖子
- 只接受论坛频道中的帖子
- 添加后会立即对该帖子执行一次检查

### 2. 批量添加多个帖子
```text
/置顶巡检 操作:批量添加 帖子列表:<多行链接或ID>
```
支持：
- 多行帖子链接
- 多个帖子 ID
- 空格 / 逗号 / 换行分隔

示例：
```text
https://discord.com/channels/服务器ID/论坛ID/帖子ID
https://discord.com/channels/服务器ID/论坛ID/帖子ID
123456789012345678
123456789012345679
```

### 3. 移除帖子
```text
/置顶巡检 操作:移除 帖子:<论坛帖子>
```
- 只会从巡检列表中移除
- 不会改变当前帖子是否置顶

### 4. 查看巡检列表
```text
/置顶巡检 操作:列表
/置顶巡检 操作:列表 论坛:<某个论坛频道>
/置顶巡检 操作:列表 页码:2
```

列表特性：
- 分页展示
- 可按论坛过滤
- 显示帖子链接
- 显示最近一次巡检状态
- 显示最近一次巡检时间

### 5. 扫描本服务器当前所有置顶帖
```text
/置顶巡检 操作:扫描本服当前置顶帖
```
系统会：
1. 扫描当前服务器所有论坛频道
2. 找出当前已经被置顶的论坛帖子
3. 列出它们的链接
4. 提供按钮：
   - 全部加入巡检列表
   - 仅加入未收录项
   - 刷新扫描

这非常适合第一次初始化巡检名单。

### 6. 导入某个论坛当前的置顶帖
```text
/置顶巡检 操作:导入本论坛当前置顶帖 论坛:<论坛频道>
```
- 如果不填 `论坛`，会尝试使用当前所在论坛或当前帖子所属论坛
- 适合只处理一个论坛的场景

### 7. 立即检查
```text
/置顶巡检 操作:立即检查
```
- 立即对当前服务器的全部巡检项执行一次检查
- 会更新列表里的最近状态
- 如果配置了报告频道，也会发送一份巡检报告

### 8. 设置报告频道
```text
/置顶巡检 操作:设置报告频道 报告频道:<文本频道>
```

### 9. 清空报告频道
```text
/置顶巡检 操作:清空报告频道
```

### 10. 开启 / 关闭巡检
```text
/置顶巡检 操作:开启
/置顶巡检 操作:关闭
```
- 只切换巡检功能开关
- 不会清空已有巡检列表

---

## 巡检状态说明

在列表和报告中，你会看到以下状态：

- `📌 正常置顶`：本次检查时帖子本来就是置顶状态
- `🔧 已自动恢复`：本次检查发现掉置顶，机器人已自动恢复
- `⚠️ 找不到帖子`：帖子不存在、无法获取，或机器人没有访问权限
- `⚠️ 目标无效`：目标已不再是论坛帖子
- `❌ 恢复失败`：机器人尝试恢复置顶失败，通常与权限或 Discord API 状态有关
- `⏳ 待检查`：已加入列表，但还没执行过完整巡检

---

## 报告频道内容

如果设置了报告频道，每次巡检都会发送一份 Embed 报告，包含：
- 服务器名称
- 触发方式（自动 / 手动）
- 巡检时间
- 总数
- 正常置顶数量
- 自动恢复数量
- 异常数量
- 失败数量
- 前若干条明细预览

---

## 数据存储位置

本模块的数据保存在：

```text
data/thread_pin_watch/{guild_id}.json
```

每个服务器一份文件，包含：
- 是否启用巡检
- 报告频道 ID
- 最近一次巡检摘要
- 巡检帖子列表
- 每个帖子的最近状态与最近检查时间

---

## 推荐使用方式

### 第一次初始化
如果一个服务器里有很多论坛频道、每个论坛通常有一个置顶帖，推荐流程：
1. 先执行：`/置顶巡检 操作:扫描本服当前置顶帖`
2. 检查扫描结果和帖子链接是否正确
3. 点击“全部加入巡检列表”或“仅加入未收录项”
4. 再执行一次：`/置顶巡检 操作:立即检查`

### 后续日常维护
- 新增一个论坛长期置顶帖：用 `添加`
- 一次补录多个帖子：用 `批量添加`
- 查看状态：用 `列表`
- 想确认当前状态：用 `立即检查`

---

## 注意事项

1. 本模块只支持 **论坛频道中的帖子**，不支持普通文本频道线程。
2. 自动巡检周期固定为 **6 小时**。
3. 如果机器人缺少论坛管理权限或编辑帖子权限，自动恢复可能失败。
4. 列表里显示的是最近一次巡检结果，不一定代表此刻的实时状态；如需实时确认请执行 `立即检查`。

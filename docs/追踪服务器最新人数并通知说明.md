# TRACK_WELCOME 模块使用指南

`TRACK_WELCOME` 是一个基于 `discord.py` 的功能插件（Cog），专门用于实时监控 Discord 服务器的人数变动，并在达到设定目标时发送自定义通知。

## 🚀 主要功能

- **人数里程碑追踪**：支持两种模式（循环增长模式 vs 固定目标模式）。
- **实时状态面板**：初始化时自动发送一个 Embed 面板，并根据设定的频率自动更新当前人数、运行时间及下一阶段目标。
- **自定义通知**：支持在通知消息中艾特多个身份组，并提供动态占位符以显示实时数据。
- **防抖逻辑**：自动处理人数在临界点反复横跳导致的重复通知问题。
- **全 DEBUG 日志**：详细记录每一次人数变动、目标更新及任务状态。

## 🛠 配置说明 (`track_config.py`)

在运行插件前，请确保配置文件包含以下参数：

| 参数 | 类型 | 说明 |
| :--- | :--- | :--- |
| `MODE` | `int` | `0`: 循环增长模式（每增加 N 人通知）；`1`: 固定目标模式（总人数达到 N 时通知）。 |
| `TARGET` | `int` | 目标数值。MODE 0 时为步进值（如 10），MODE 1 时为目标总数（如 10000）。 |
| `NOTI_CHANNEL` | `int` | 发送初始化面板和达成通知的频道 ID。 |
| `NOTI_IDENTITY_GROUP` | `str` | 需要提现的身份组 ID，多个 ID 用英文逗号 `,` 分割。 |
| `CHANGE_TIME` | `int` | 状态面板自动刷新频率（秒），默认 `60 * 60`。 |
| `NOTI_TEXT` | `str` | 目标达成时的消息模板，支持占位符。 |

## 📝 消息占位符

在 `NOTI_TEXT` 文本中，你可以使用以下占位符来动态显示数据：

- `{{current_members}}`：当前服务器的总人数。
- `{{welcome新增人数}}`：本次里程碑设定的目标数值。
- `{{实际增长人数}}`：自机器人本次启动以来，服务器人数的净增长值。

**示例：**
```python
NOTI_TEXT = """
🎉 恭喜！服务器人数已达到 {{current_members}}！
本次我们达成了新增 {{welcome新增人数}} 人的目标。
自追踪任务启动以来，已有 {{实际增长人数}} 位新伙伴加入！
"""
```

## 🖥 状态面板说明

任务启动后，会在 `NOTI_CHANNEL` 发送一个绿色 Embed 消息，包含：
1. **当前人数**：服务器实时人数。
2. **目标人数**：下一阶段或最终需要达到的数值。
3. **运行模式**：当前任务的运行逻辑。
4. **已运行时间**：从任务初始化至今经过的时长。
5. **最后更新时间**：面板最后一次编辑的时间戳。

## 🕹 管理命令

- `!trackstatus`：管理员可随时使用此命令手动查看当前的追踪详情（不影响自动更新的面板）。

## ⚠️ 注意事项

1. **Intents 权限**：Bot 必须开启 `Server Members Intent` 才能获取准确的人数及监听成员变动。
2. **消息编辑权限**：Bot 必须拥有在目标频道发送消息及编辑自己消息的权限。
3. **初始化触发**：脚本会在 Bot 启动后，发生第一次成员变动（加入/离开）时正式初始化并发送状态面板。
4. **ID 格式**：配置文件中的所有 ID（频道、身份组、服务器）请确保为整数（int）或可转为整数的字符串。
